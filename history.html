Hashing was Introduced by Salakhutdinov and Hinton, semantic hashing uses binary code to agree with the semantic similarity via the hamming distance between codewords. To elaborate, everything in the database would be regarded as binary code. Semantic hashing uses compact binary code. It is set up this way so that identical items would have identical codewords. The retrieval for semantic hashing is extraordinarily quick, approximately millions of queries every second. However, the key to make all of this work is to provide good code. Said good code has to meet the criteria, (a) can be easily calculated so that it could be used for the novel input (b) calls for a minimal number of bits for the dataset code & (c) direct equivalent items to their matching counterparts.
